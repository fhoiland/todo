# Repository guide

This guide defines how the repository should be read.

## Current source of truth
Use these files first:

- `README.md`
- `docs/current-state.md`
- `docs/architecture-current.md`
- `docs/node-red-alarm-flow-current.md`
- `docs/node-red-devices-object-current.md`
- `docs/security.md`
- `docs/deployment-proxmox-foundation.md`
- `docs/todo-next-phase.md`

## Historical / milestone notes
These files are kept as project history and step-by-step documentation:

- `docs/progress-log.md`
- `docs/2026-03-29-node-red-alarm-hardening.md`
- `docs/2026-03-29-step7-notification-format.md`
- `docs/2026-03-29-step8-ntfy-security-and-docker-networking.md`
- `docs/node-red-alarm-flow.md`
- `docs/node-red-devices-object.md`
- `docs/architecture.md`
- `TODO.md`

Some of the historical files describe earlier stages of the project and may not exactly match the latest implementation.

## Reading order for new deployments

1. `docs/current-state.md`
2. `docs/architecture-current.md`
3. `docs/deployment-proxmox-foundation.md`
4. `docs/security.md`
5. `docs/node-red-alarm-flow-current.md`
6. `docs/node-red-devices-object-current.md`
7. `docs/todo-next-phase.md`

## Purpose
This structure keeps useful history without forcing future projects to guess which document reflects the real current setup.
